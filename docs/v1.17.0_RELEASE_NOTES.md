# xAI SDK Go v1.17.0 Release Notes

**Release Date:** June 19, 2026

## Focus: Python SDK v1.17.0 Parity

This release ports the public API additions from the official Python SDK v1.17.0, focused on Files API public URLs, server-side file list filtering, file-ID inputs for image and video generation, and Files API storage options for generated assets.

## Added

### Protobuf and Service Surface Parity
- Added Files API public URL request/response messages and `CreatePublicUrl` / `RevokePublicUrl` RPCs.
- Added `ListFilesRequest.filter` and public URL metadata on file responses.
- Added image/video generation `storage_options`, generated asset file output metadata, storage errors, and public URL error metadata.
- Added Files API `file_id` references for image inputs, video inputs, and reference image inputs.

### Go Client Conveniences
- Added `files.Client.CreatePublicURL()` and `files.Client.RevokePublicURL()`.
- Added `files.ListOptions.Filter`.
- Added `files.StorageOptions` plus public URL option helpers for generated asset persistence.
- Added image request helpers for single and multi-reference file-ID inputs and storage options.
- Added video generation and extension helpers for image/video file IDs, reference image file IDs, and storage options.
- Added image/video response accessors for file output, storage errors, public URLs, and public URL errors.

## Fixed

- Removed the local `google/rpc/status.proto` shadow and configured Buf to keep Google API imports canonical, preventing duplicate protobuf descriptor registration.
- Fixed REST request handling so pre-marshaled protobuf JSON payloads are sent as raw JSON instead of being encoded as byte arrays.
