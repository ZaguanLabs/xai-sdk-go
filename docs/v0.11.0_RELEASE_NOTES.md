# xAI SDK Go v0.11.0 Release Notes

**Release Date:** March 12, 2026

## 🎯 Focus: Python SDK v1.8.1 Public-Surface Parity

This release brings the Go SDK to public-surface parity with the official Python SDK `v1.8.1` sync client surface. The focus of the release is convenience and ergonomic parity: chat helpers, stored completions, batch/video workflows, collections/document conveniences, and top-level client accessors now align closely with the official Python source.

## ✨ New Features

### Chat Client Wrapper

The top-level client now exposes a higher-level chat wrapper:

```go
client, _ := xai.NewClientFromEnvironment()

chatClient := client.Chat()
req := chatClient.Create("grok-3",
    chat.WithMessage(chat.User(chat.Text("Hello"))),
)

resp, err := req.Sample(context.Background(), chatClient)
```

If you need the underlying generated stub, you can still use:

```go
raw := client.RawChat()
_ = raw
```

### Stored Completion Helpers

Stored chat completions now have first-class SDK helpers:

```go
responses, err := client.Chat().GetStoredCompletion(ctx, "response_123")
deletedID, err := client.Chat().DeleteStoredCompletion(ctx, "response_123")
```

### Batch Chat Utilities

The Go chat request layer now includes Python-style multi-output helpers:

```go
responses, err := req.SampleBatch(ctx, client.Chat(), 3)
stream, err := req.StreamBatch(ctx, client.Chat(), 2)
responses, err := req.DeferBatch(ctx, client.Chat(), 3, 5*time.Minute, 100*time.Millisecond)
_ = stream
```

### Structured Parsing with Raw Response Access

Structured parsing can now retain the raw response:

```go
var parsed map[string]interface{}
resp, err := req.ParseWithResponse(ctx, client.Chat(), &parsed)
_ = resp
```

### Batch Client Ergonomics

The batch client now accepts chat requests directly:

```go
err := client.Batch().Add(ctx, batchID, req1, req2)
requests, next, err := client.Batch().ListBatchRequests(ctx, batchID, nil)
results, next, err := client.Batch().ListBatchResults(ctx, batchID, nil)
_, _, _ = requests, next, results
```

### Video Convenience Helpers

The video client now matches the Python deferred workflow more closely:

```go
start, err := client.Video().Start(ctx, "Generate a short clip", "grok-video", nil)
status, err := client.Video().Get(ctx, start.RequestId)
videoResp, err := client.Video().GenerateAndPoll(ctx, "Generate a short clip", "grok-video", nil)
_, _ = status, videoResp
```

### Collections Search and Upload Conveniences

The collections client now exposes high-level search and upload workflows:

```go
searchResp, err := client.Collections().Search(ctx, "sdk parity", []string{"col_123"}, &collections.SearchOptions{
    Instructions:  "Prefer recent SDK documentation",
    RetrievalMode: "semantic",
})

doc, err := client.Collections().UploadDocument(ctx, "col_123", "notes.txt", []byte("hello"), &collections.UploadDocumentOptions{
    WaitForIndexing: true,
})
_, _ = searchResp, doc
```

## 🔧 Parity Highlights

### Top-Level Client Accessors

The main client now provides parity-sensitive accessors aligned with the Python naming:

- `Chat()`
- `RawChat()`
- `Image()`
- `Images()`
- `Tokenize()`
- `Tokenizer()`
- `Batch()`
- `Video()`

### Chat Request Enhancements

Additional parity features are included in the chat request layer:

- `developer` role support
- `tool_call_id` support
- `max_turns`
- `include` options
- `batch_request_id`
- web search `user_location`

### Collections / Documents

The following convenience methods are now available on the collections client:

- `Create()`
- `Get()`
- `List()`
- `Update()`
- `Delete()`
- `Search()`
- `UploadDocument()`
- `GetDocument()`
- `ListDocuments()`
- `UpdateDocument()`
- `DeleteDocument()`
